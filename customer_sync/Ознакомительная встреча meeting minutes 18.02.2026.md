## Протокол встречи

### Ответы на вопросы

1. Интеграция в основной репозиторий
Целевым вариантом размещения кода является основной репозиторий проекта, а не форк. В связи с этим при разработке необходимо придерживаться принятого в проекте стайл-гайда.

2. Структура Pull Request'ов
Для каждого устройства создаётся отдельный Pull Request. Необходимо предварительно изучить существующую структуру проекта и способ описания устройств в репозитории.

3. Реализация клавиатуры
Клавиатура должна поддерживать раскладку QWERTY. Требуется реализовать перехват событий с хостовой (физической) клавиатуры. Основной приоритет — корректная передача текстового ввода. Поддержка сочетаний клавиш на данном этапе остаётся открытым вопросом.

4. Протокол взаимодействия клавиатуры
Для выбора протокола необходимо проанализировать реализацию существующих устройств в RISC-V Pipe Simulator (RIPES). Эмуляция аппаратных шин (USB и др.) не является приоритетной задачей.

5. Замена RGB-монитора
Реализация отдельного RGB-монитора признана нецелесообразной, поскольку уже существующая LED-матрица поддерживает отображение в RGB. Принято решение заменить монитор на другое устройство. Контрольная точка — конец следующей недели; при необходимости направить напоминание.
