### Не выполненные задачи

**Задача 1 — тесты через Selenium не реализованы.**

Ripes поддерживает сборку под WebAssembly (Qt for WebAssembly). В этом случае весь UI рендерится через WebGL/Canvas напрямую из C++/Qt, без генерации DOM-элементов. Selenium работает с DOM-деревом браузера: он ищет элементы по тегу, id, классу и т.п. Поскольку Qt/WASM не создаёт никакого DOM (вся страница — один <canvas>), Selenium не может взаимодействовать с виджетами приложения: нельзя кликнуть кнопку, прочитать значение поля, проверить состояние виджета.

Альтернативы для UI-тестирования Qt/WASM: Playwright с перехватом canvas-событий (ненадёжно), либо тестирование на уровне Qt-слоя через QTest без браузера.
